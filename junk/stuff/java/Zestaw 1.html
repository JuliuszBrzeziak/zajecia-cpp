Instalacja JDK i JavaFX

Aby tworzyć oprogramowanie w języku Java należy zainstalować środowisko Java Development Kit (JDK, Java Standard Edition Software Development Kit). Istnieją dwie główne implementacje języka programowania Java - Oracle JDK oraz OpenJDK (wolnodostępna i otwarta, ang. free software and open source). Obecnie OpenJDK jest implementacją referencyjną i posiada wszystkie istotne elementy Oracle JDK.

JavaFX to nowoczesny zbiór narzędzi służący między innymi do tworzenia graficznego interfejsu użytkownika (GUI) w aplikacjach napisanych w Javie. Począwszy od wersji JDK 11, JavaFX nie wchodzi w jej skład i dostępny jako osobny pakiet rozwijany w ramach projektu OpenJFX będącego częścia OpenJDK.

System operacyjny Windows

  • Pobrać ze strony https://jdk.java.net/12/ odpowiedni pakiet OpenJDK 12
  • Rozpakować archiwum np. w katalogu C:\Program Files\Java\
  • Dodać do zmiennej środowiskowej PATH ścieżkę do katalogu bin (np. C:\Program Files\Java\jdk-12.0.2\bin)
  • Ustawić zmienną środowiskową JAVA_HOME na ścieżkę do instalacji JDK (np. C:\Program Files\Java\jdk-12.0.2)
  • Pobrać ze strony https://gluonhq.com/products/javafx/ odpowiedni pakiet JavaFX 13
  • Rozpakować archiwum np. w katalogu C:\Program Files\Java\
  • Ustawić zmienną środowiskową PATH_TO_FX na ścieżkę do katalogu lib pakietu JavaFX (np. C:\Program Files\Java\javafx-sdk-13\lib)

Zmienną środowiskową dla użytkownika można ustawić poprzez Panel sterowania: Control Panel -> User Accounts -> User Accounts -> Change my enviroment variables lub w powłoce cmd.exe komendą set.

System operacyjny Mac OS

  • Pobrać ze strony https://jdk.java.net/12/ odpowiedni pakiet OpenJDK 12
  • Jako administrator rozpakować archiwum w katalogu /Library/Java/JavaVirtualMachines/

System operacyjny Linux

Większość dystrybucji udostępnia pakiety OpenJDK oraz OpenJFX.

Aby sprawdzić czy instalacja przebiegła poprawnie można wywołać

java -version
javac -version

Zadanie 1.

Na podstawie programów Hello world przedstawionych na wykładzie, należy napisać program Hello.java, który zawiera klasę publiczną Hello, oraz

  • Dodać konstruktor.
  • Dodać statyczne wyrażenie inicjujące.
  • Domyślnie, każda klasa rozszerza klasę java.lang.Object.
  • Zaimplementować publiczną funkcję statyczną public static void main(String[] args), która wypisze argumenty programu i utworzy obiekt tej klasy.
  • Konstruktor, statyczne wyrażenie inicjujące i funkcja main() mają wyświetlać odpowiedni komunikat w konsoli.

Kompilator i środowisko Javy jest zainstalowane na komputerach studenckiej pracowni pod linuxem.
Kompilacja:

javac Hello.java

Uruchomienie byte-code’u:

java Hello

Proszę zwrócić uwagę na rozszerzenia plików, bądź ich brak.

Uwaga: Java od wersji 12 pozwala na bezpośrednie uruchomienie aplikacji jedno-plikowych z pominięciem kopmpilacji:

java Hello.java

Zadanie 2. HelloFX

Napisać program HelloFX.java, który wyświetla okno wraz z przyciskiem i polem tekstowym używając pakietu javafx. Naciśnięcie przycisku powinno zmodyfikować tekst w polu tekstowym. Zaimplementować klasę HelloFX, która rozszerza klasę javafx.application.Application Dla aplikacji JavaFX metoda statyczna main() nie jest niezbędna, ale do dobrych praktych należy jej implementacja, która wywołuje jedynie launch(args). Aplikacja powinna korzystać z takich klas pakietu JavaFX jak Text, Button, TextField, i układu VBox.

Zaimportować odpowiednie pakiety/klasy.

Przykładowa aplikację można znaleźć w dokumentacji.
Tutaj można znaleźć dokumentację JavaFX 13.

Do kompilacji i uruchomienia tego programu wymaga jest Java 8 lub nowsza. Na komputerach w pracowni można użyć kompilatora javac8 i JVM java8.

Zadanie 4. Print

Napisać i skompilować program wypisujący zawartość na ekran konsoli dowolnego pliku, którego nazwa podana jest jako argument z linii komend.

Wskazówka: Proszę wykorzystać odpowiednie klasy wymienione w Zadaniu 1. Niektóre metody zwracają wyjątki(exceptions), taki fragment kodu należy objąć komendą try { ... }, a następnie złapać wyjątki przy pomocy komendy catch(Exception e) { ... }. Pomocny może się okazać poniższy schemat kodu:

try
{
    /* Tu nalezy odczytac i wypisac
       zawartosc jakiegos pliku */
    ...
}
catch(FileNotFoundException e)
{
    System.out.println("Plik nie zostal znaleziony");
}
catch(IOException e)
{
    System.out.println("Blad wejscia/wyjscia");
}

Wskazówka: Tablice bajtów deklarujemy w następujący sposób:

byte[] data;

a definiujemy (automatycznie alokuje pamięc i inicjalizuje do zer, porównaj z C/C++) tak:

data = new byte[len];

W Javie tablice są obiektami. Ich długość można odczytać z pola length, np. data.length.

Zadanie 5. Input/output (binary files)

Napisać program CopyFile, będący modyfikacją programu Print. Program ma kopiować zawartość jednego pliku do drugiego (pliki binarne), nazwy plików są podane jako argumenty z linii komend. Zaimportować pakiet java.io. Użyć klas FileInputStream i FileOutputStream oraz metod read i write. Jako bufor pośredni użyć tablicy bajtów.
Ze względu na ograniczony rozmiar tablicy operacje należy umieścić w pętli. Obsłużyć odpowiednie wyjątki.

Zadanie 6. Input/output (buffered)

Napisać program PrintFile, który wyświetla zawartość pliku tekstowego linijka po linijce i je numeruje. Nazwa pliku jest podana jako argument z linii komend. Zaimportować pakiet java.io. Użyć klas FileReader i BufferedReader. Obsłużyć odpowiednie wyjątki.
